# 发卷小程序实施计划

> 截至 `2026-04-12`：`backend + frontend + database + miniapp` 首轮主线已基本打通，当前重点已转为“发券联调验收、真实微信支付参数补齐、真机全链路验收、安全/环境收尾”。本文档用于持续勾选完成状态。

**目标**
- 落地一套可上线扩展的发券系统，覆盖后台配置、售券、支付后发券、用户券管理、ERP 核销，并逐步接入小程序与微信能力。

**当前技术栈**
- `backend/`：`.NET 8` + `ASP.NET Core Web API` + `EF Core`
- `frontend/`：`Vue 3` + `TypeScript` + `Vite`
- `database/`：`SQL Server 2008 R2`
