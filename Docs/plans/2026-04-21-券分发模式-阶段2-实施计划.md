# 券分发模式（阶段 2）实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让单张售卖券与商品券都能在小程序 `mall` 页展示、下单、支付并在支付成功后发券，同时为商品券补“待履约 / 待 ERP 处理”状态。

**Architecture:** 以“统一订单模型 + 统一支付入口 + `mall` 聚合接口 + 小程序商城多区块展示”为主线推进。订单层把“券包订单”和“单张券订单”统一为一个 `CouponOrder` 模型；支付成功后由 `OrderPaymentService` 按订单来源分流发券；商品券只落“待履约”状态，不做 ERP 对接。

**Tech Stack:** .NET 8 / EF Core 8 / SQL Server / xUnit + InMemory / uni-app + Vue 3 / 微信小程序支付链路

**设计文档：** `Docs/specs/2026-04-21-券分发模式-阶段2设计.md`
