# 2026-04-19 首页轮播 UI 调整方案

## 一、背景

上一轮把首页 banner 改为"标题 + 副标题 + CTA 按钮 + 右侧插画"的样式，但对比用户提供的目标图（局部橙色 banner，带清晰指示器 + 3D 钱包插画），存在明显差距。结合用户反馈，本次聚焦 **banner 内部调整**，不动 hero 其他区域。

## 二、用户需求确认（本轮已达成共识）

| 项 | 决策 |
|---|---|
| GO> 显示为 `GO&gt;` | 明确 bug，立即修 |
| banner 标题字号 | 缩小 |
| swiper 指示器 | 移到 banner 外部 / 换更清晰的点 |
| CTA 按钮 | 细化（padding、阴影、字号） |
| banner 数量 | 由后端 API 返回决定，前端不写死 |
| 插画素材 | 暂不换图，先把布局调对 |
| hero 区域 | 不动 |

## 三、已完成改动（本次 session 已落地）

### 3.1 template

- `swiper` 去掉 `indicator-dots` / `indicator-color` / `indicator-active-color`；改为 `:circular="banners.length > 1"` / `:autoplay="banners.length > 1"`（单条不循环不自动播）；加 `@change="onBannerChange"`。
- swiper 后追加自定义 dots：`<view class="banner-dots" v-if="banners.length > 1">`，内部按 `banners` 渲染 `.banner-dot`，高亮态加 `.banner-dot--active`。
- 两个 entry-card 的 `<text class="entry-go-arrow">&gt;</text>` + 前一个 `<text class="entry-go-text">GO</text>` 合并为单个 `<text class="entry-go-text">GO ›</text>`（Unicode U+203A，彻底绕开 HTML 实体问题）。

### 3.2 script

- 新增 `const currentBanner = ref(0)`
- 新增 `function onBannerChange(e) { currentBanner.value = e?.detail?.current ?? 0 }`

## 五、兼容性 & 影响面

- **后端 API 契约不变**：继续使用 `banners[]`（id / title / subtitle / ctaText / illustrationUrl / linkUrl）
- **本地开发兜底数据**：保留当前 2 条占位，仅在 API 返回空时显示
- **暗色底（默认/橙/红）用白点，浅色底（light/candy）用深色点**，对比度保证

## 六、预期效果

- 标题"购买券包"不再撑满半屏宽度
- 轮播圆点在 banner 下方 16rpx 处，清晰可见
- CTA 按钮视觉权重降低，不与标题抢戏
- 根据 `banners.length` 自动决定是否启用循环/自动播和指示器
