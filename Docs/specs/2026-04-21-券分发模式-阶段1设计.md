# 券分发模式（Coupon Distribution Mode）· 阶段 1 设计

- 日期：2026-04-21
- 作者：Claude（与 ssfnxk007 结对）
- 状态：设计评审中
- 关联：阶段 2（单张下单 + 商品券核销）将另写 spec

---

## 1. 背景与问题

小程序首页"免费领取"区目前无差别展示所有 `IsEnabled=true` 的 `CouponTemplate`。这与实际业务诉求不符——业务上有几类券：

- **免费券**：首页直接领取
- **付费单卖券**（未来要做）：小程序商城单张购买
- **商品券**（未来要做）：代表某个具体商品，买下后到线下 ERP 核销取货
- **组包券**：只用来组装 `CouponPack` 付费卖，**不该**在小程序任何免费入口暴露

当前没有字段区分，所有 enabled 的券全上首页，运营无法放心把付费场景的券建进来。

---

## 2. 阶段划分与本 spec 范围

### 阶段 1（本 spec）· 数据层 + 后台录入 + 小程序展示分区

1. `CouponTemplate` 新增 `DistributionMode`、`SalePrice` 字段
2. EF Core 迁移 + 历史数据默认值
3. 后台券模板创建/编辑 UI 支持新字段
4. 小程序端所有公开列表 / 商品推荐场景的查询加 `DistributionMode == FreeClaim` 过滤
5. 后台 / 前端的最小测试覆盖

**立即交付价值：** 首页"免费领取"区立刻变干净——PackOnly 和 PaidStandalone 的券一张也不会露出。后台能完整建模各类型券的分发属性。

### 阶段 2（后续独立 spec，不在本 spec 范围）

- `CouponOrder` 扩展：加 `CouponTemplateId?`，`CouponPackId?` 变可空，两者二选一
- 单张券下单接口（miniapp controller + payments controller 适配）
- `mall` 页扩展"单张券"区块
- 商品券核销流程（目前**仅记录"待 ERP 发货"状态**，不做 ERP API 对接）
- ERP 发货对接作为阶段 2 之后的独立 feature，不在这次规划内

## 4. 数据模型改动

### 4.1 新枚举

`backend/src/FaJuan.Api/Domain/Enums/CouponDistributionMode.cs`

```csharp
namespace FaJuan.Api.Domain.Enums;

public enum CouponDistributionMode
{
    FreeClaim = 0,        // 免费领取：首页公开展示，用户免费领
    PaidStandalone = 1,   // 单张售卖：小程序商城单独买（阶段 2 开放入口）
    PackOnly = 2,         // 仅组包：只能通过购买 CouponPack 获得
}
```

### 4.2 `CouponTemplate` 实体扩展

`backend/src/FaJuan.Api/Domain/Entities/CouponTemplate.cs` 新增：

```csharp
public CouponDistributionMode DistributionMode { get; set; } = CouponDistributionMode.FreeClaim;
public decimal? SalePrice { get; set; }
```

**字段语义：**
- `DistributionMode`：分发路径，3 选 1，默认 `FreeClaim`（旧数据兼容）
- `SalePrice`：单张售价（元，保留两位小数）。仅当 `DistributionMode == PaidStandalone` 时有业务意义且必填；其他模式下必须为 null

> **不新增 `ProductId`：** 项目已有 `CouponTemplateProductScope` 处理券与商品的多对多关联。`TemplateType == Product` 的商品券通过这张关联表指向目标商品（现状保留），阶段 2 的核销服务读它拿兑换目标。

### 4.3 EF 迁移 `AddCouponDistributionMode`

| 列 | 类型 | 约束 |
|---|---|---|
| `DistributionMode` | `INT` (SQL Server 存储 enum) | NOT NULL DEFAULT 0 |
| `SalePrice` | `DECIMAL(18,2)` | NULL |

`Up()` 内对历史行补 `DistributionMode = 0`（FreeClaim），保留现有首页展示行为不回归。

### 4.4 约束一致性

在 `Admin` 层做应用级校验（不依赖 DB check constraint，兼容 SQL Server 2008 R2）：

- `DistributionMode == PaidStandalone && (SalePrice == null || SalePrice <= 0)` → 返回 400 "单张售卖券必须填写售价，且大于 0"
- `DistributionMode != PaidStandalone && SalePrice != null` → 后端强制 `SalePrice = null` 入库（直接忽略前端传来的非法值，不报错）。前端编辑页切换分发模式时应本地自动清空输入框，保持 UI 和后端行为一致

## 5. 后端改造

### 5.1 CouponTemplatesController

`backend/src/FaJuan.Api/Controllers/CouponTemplatesController.cs`：

- **List DTO** / **Detail DTO**：新增 `DistributionMode`（int）、`SalePrice`（decimal?）两个字段回显
- **Create Request** / **Update Request**：新增同名字段
- **Validate**：
  - `DistributionMode` 必填（枚举值 0/1/2 合法）
  - `PaidStandalone` 时 `SalePrice > 0`，否则 400
  - 非 `PaidStandalone` 时后端强制 `SalePrice = null` 入库

### 5.2 MiniAppController 小程序端过滤

给**所有公开展示**的 CouponTemplate 查询加 `x.DistributionMode == CouponDistributionMode.FreeClaim` 过滤：

| 位置 | 用途 | 改动 |
|---|---|---|
| `MiniAppController.cs:106-110` | 首页「免费领取」+「新人专享」（directCoupons） | 加 FreeClaim 过滤 |
| `MiniAppController.cs:275-281` | 商品详情「相关券」 | 加 FreeClaim 过滤 |
| `MiniAppController.cs:302-305` | 商品详情「推荐券」 | 加 FreeClaim 过滤 |
| `MiniAppController.cs:411` | 券详情 `GetCouponTemplateDetail` | **不加**（宽松策略，用户知道 URL 也能访问）|

**不过滤的地方**（保持现状）：
- 券包详情 items（买包后看到包里所有券，包含 PackOnly 券）
- 用户"我的券"列表（已持有的不受分发模式限制）

## 7. 数据迁移

1. 迁移 `Up()` 里 `AddColumn` 时带 `defaultValue: 0` → 旧数据全部归类为 `FreeClaim`，**保留现有首页展示行为**。用户虽说可清空，但默认值方案更稳。
2. 旧数据的 `SalePrice` 默认 null，符合"非 PaidStandalone 应为 null"的约束。
3. 若你选择手动清空全部券模板：不影响本迁移的正确性，只是种子行少。

## 8. 测试策略

### 8.1 后端单元测试（`backend/tests/FaJuan.Api.Tests/`）

- `CouponTemplatesControllerTests`（新建）
  - `Create` 时 `PaidStandalone` 不带 `SalePrice` → 返回 400
  - `Create` 时 `PaidStandalone` 且 `SalePrice = 0` → 返回 400
  - `Create` 时 `FreeClaim` 带 `SalePrice=5` → 入库后 `SalePrice == null`（后端容错）
  - `Update` 语义同上
- `MiniAppControllerTests`（如已存在就扩展，否则新建覆盖首页部分）
  - 有 3 张券：FreeClaim / PaidStandalone / PackOnly 各一张 → `GetHome` 返回的 directCoupons 只有 FreeClaim 那一张

### 8.2 前端手测

- 后台创建 `FreeClaim` 券 → 首页免费领取区看到
- 后台创建 `PaidStandalone` 券（带 SalePrice）→ 首页**看不到**、mall 也看不到（阶段 2 才会加入口）
- 后台创建 `PackOnly` 券 → 首页**看不到**，在券包编辑器里可选作为 PackItem
- 后台把 `PaidStandalone` 的 SalePrice 置空 → 返回校验错误
