# 小程序下拉刷新与上拉加载设计

- 日期：2026-04-21
- 范围：`miniapp/coupon-miniapp/`
- 目标：为 tab 页与列表页补齐下拉刷新 + 上拉加载更多能力，视觉与主题统一，可复用

## 1. 背景

小程序 14 个页面目前都没有配下拉刷新，也没有分页加载。页面 `pages.json` 使用 `navigationStyle: "custom"` + `cm-nav-spacer` 顶栏占位，主题系统通过 `.cm-page` 的 CSS 变量切换（default / light / candy / orange / red）。

用户反馈：在卡包、订单等列表页无法主动刷新数据，且订单增多后无分页入口。

## 2. 决策摘要

| 决策 | 选择 | 理由 |
| --- | --- | --- |
| 刷新样式 | 自定义（非小程序原生三点动画） | 与主题一致、可加 logo |
| 容器实现 | `scroll-view` + `refresher-enabled` | uni-app 标准做法，支持 refresher slot |
| 加载更多触发 | `scroll-view` 的 `@scrolltolower` | 与下拉刷新同容器，状态集中 |
| 复用方式 | 新建 `CmPullRefresh` 组件 + `use-list-pagination` composable | 避免每页重写状态机 |
| 接入范围 | 4 个 tab 页 + 2 个列表页，共 6 页；详情页暂不接入 | 聚焦用户明确提到的范围，详情页无列表/刷新需求 |

## 3. 组件设计：`components/CmPullRefresh.vue`

### 3.1 Props

| 名称 | 类型 | 默认 | 说明 |
| --- | --- | --- | --- |
| `refreshing` | Boolean | false | 受控下拉刷新状态；true 时 refresher 保持打开 |
| `loadingMore` | Boolean | false | 底部加载中状态，显示 loading 条 |
| `noMore` | Boolean | false | 已加载完毕，底部显示"已到底啦" |
| `showLoadMore` | Boolean | false | tab 页无分页时设为 false，彻底隐藏底栏 |
| `scrollTop` | Number | 0 | 可选，外部受控滚动位置（tab 切换保留） |

### 3.2 Events

- `refresh` — 用户下拉到阈值并松手时触发；外部应在完成后置 `refreshing=false`
- `loadmore` — 滚动到距底部约 80rpx 时触发；外部应在 `loadingMore` 为 false 且 `noMore` 为 false 时才响应，避免重入
- `scroll` — 透传 `scroll-view` 的滚动事件，便于外部保存位置

### 3.3 Slots

- 默认 slot — 页面原本的内容
- 无需对外暴露 refresher slot，refresher UI 封装在组件内部

### 3.4 视觉规格

**下拉刷新头**
- 高度：100rpx；背景透明
- 中心：logo.png，36rpx × 36rpx
- 下方文案：`下拉刷新` → `松手刷新` → `正在刷新…`，随 refresher 距离切换
- 颜色：`var(--cm-theme-primary-strong)`，4 种主题自动适配
- refreshing 时 logo 匀速旋转（1s / 360deg）

**底部加载态**
- 高度：80rpx；居中
- loadingMore：24rpx loading 圆点 + "加载中…"
- noMore：仅文案"已到底啦"，颜色 `var(--cm-theme-primary-soft)`

### 3.5 关键实现点

```
<scroll-view
  scroll-y
  :refresher-enabled="true"
  refresher-default-style="none"
  :refresher-triggered="refreshing"
  @refresherrefresh="onRefresherRefresh"
  @scrolltolower="onScrollToLower"
  :style="{ height: '100vh' }"
  lower-threshold="80">

  <view slot="refresher" class="cm-pull-refresher">
    <image src="/static/logo.png" :class="['cm-pull-logo', refreshing && 'is-spinning']" />
    <text class="cm-pull-text">{{ refreshText }}</text>
  </view>

  <slot />

  <view v-if="showLoadMore" class="cm-pull-footer">
    <template v-if="loadingMore">…加载中…</template>
    <template v-else-if="noMore">已到底啦</template>
  </view>
</scroll-view>
```

## 4. Composable：`composables/use-list-pagination.js`

列表页共用的分页状态机，避免每个页面重复写。

### 4.1 接口

```
const {
  items,          // ref<Array>
  refreshing,     // ref<boolean>
  loadingMore,    // ref<boolean>
  noMore,         // ref<boolean>
  pageIndex,      // ref<number>
  pageSize,       // number (常量，默认 10)
  onRefresh,      // 函数：重置到第一页
  onLoadMore,     // 函数：加载下一页
  reload          // 函数：同 onRefresh，提供语义化别名
} = useListPagination(fetcher, { pageSize = 10 })
```

### 4.2 fetcher 契约

外部传入的 `fetcher(query)` 必须返回：

```
{
  items: Array,
  pageIndex: number,
  totalPages: number,
  totalCount: number
}
```

对齐现有 `fetchOrderList` 的返回结构，不需要改后端契约。

### 4.3 核心逻辑

- `onRefresh`：`refreshing=true` → `pageIndex=1` → 调 fetcher → 替换 `items` → `noMore = pageIndex >= totalPages` → `refreshing=false`
- `onLoadMore`：若 `loadingMore || noMore` 则 return → `loadingMore=true` → `pageIndex+=1` → 调 fetcher → 追加 `items` → 更新 `noMore` → `loadingMore=false`
- 异常：fetcher 抛错时 `uni.showToast` 提示"加载失败"，并回滚 `pageIndex`

### 5.1 阶段一：tab 页（只下拉刷新，无分页）

| 页面 | 刷新内容 | showLoadMore |

## 7. 风险与已知约束

1. **scroll-view 接管页面滚动**
   当前所有 cm-page 靠 body 自然滚动。改用 `scroll-view` 后必须显式给 `height: 100vh`。已验证页面的 hero/nav-spacer 都是 flow 布局，不依赖 body-scroll，迁移风险可控。
2. **verify-record 后端分页**
   若 `api/miniapp.js` 中核销记录接口尚未返回 `pageIndex/totalPages`，该页必须先等后端对齐，否则上拉加载不可用。阶段二开始前需先核查。
3. **tab 切换保留滚动位置**
   `scroll-view` 不自动保留滚动位置。若产品期望"返回 tab 时回到之前位置"，需把 `scroll-top` 提到 pinia session store 保存。当前范围不做，按默认"回到顶部"。
4. **swiper 与 scroll-view 嵌套**
   `pages/index/index.vue` 有 banner `<swiper>`。uni-app 中 swiper 放在 vertical scroll-view 内部是支持的，但需验证手势冲突。
5. **profile 页的 tab 栏高度**
   tabbar 由小程序原生渲染，不在 scroll-view 内，`height: 100vh` 的 scroll-view 底部会被 tabbar 遮挡。需用 `calc(100vh - env(safe-area-inset-bottom))` 或在父容器用 `cm-page` 原本的 `min-height` 方案——具体数值在实现时按真机调。

## 8. 不在本次范围

- 详情页（coupon/detail、product/detail、coupon-pack/detail、sale-coupon/detail、activity/detail、order/result）不加刷新
- 骨架屏 / 首次加载占位 UI
- 刷新失败的重试 UI（仅 toast 提示）
- 弱网场景的下拉提示优化

## 9. 验收标准

- 4 个 tab 页下拉均能触发接口重新加载，logo 旋转动画正常
- `pages/order/list.vue` 下拉重置到第一页；上拉能加载下一页直到 `noMore=true` 显示"已到底啦"
- 所有主题（default / light / candy / orange / red）下刷新头颜色一致使用主题色
- 网络失败时 toast 提示，界面不卡在 `refreshing=true`
- 核销记录页根据后端对齐情况，能下拉 + 上拉或至少下拉刷新
