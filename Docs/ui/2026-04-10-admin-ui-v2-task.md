# 后台 UI 深收口任务（V2）

日期：`2026-04-10`

## 目标
- 将后台管理前端视觉风格统一收口到 `Docs/ui/admin-preview-v1.html` 的方向。
- 本轮仅处理 `frontend/` 的 UI / UX，不处理 `backend/`、数据库、接口兼容性。
- 去掉页面中的“模拟感”展示，不在 UI 上出现“模拟登录 / 模拟支付 / 联调专用 / 测试用”等字样。
- 禁止使用 emoji 图标，统一改为纯样式图标体系。

## 范围
- 仅允许修改：`frontend/`
- 参考稿：`Docs/ui/admin-preview-v1.html`
- 重点页面：
  - `商品管理`
  - `券模板管理`
  - `用户券`
  - `核销中心`
- 共享层优先：
  - 后台整体布局
  - 侧栏导航
  - 顶栏
  - 卡片
  - 筛选工具栏
  - 表格
  - 弹窗
  - 按钮 / 输入框 / 标签 / 空状态

## 不做项
- 不改后端接口、不改数据库、不改 EF 迁移、不改 SQL。
- 不处理小程序 UI。
- 不新增真实业务功能，只优化现有前端展示与交互层级。
- 不在本轮引入新的第三方 UI 框架。

## 核心约束
- 后台 UI 不得再使用 emoji 图标。
- 页面文案中不要明显出现“模拟”字样；如功能暂时仍是演示逻辑，也要用正式业务表达包装。
- 保持现有功能入口与权限控制逻辑不变，只做前端视觉和交互收口。
- 优先改共享样式与共享布局，避免每页单独写一套风格。

## 共享视觉基线

### 1. 布局
- 侧栏采用深色沉稳风格，强调分组、激活态、层级感。
- 顶栏采用浅色半透明卡片感，减少厚重边框。
- 内容区留白增大，页面信息块之间保持稳定节奏。

### 2. 图标
- 全部使用纯样式图标、字母图标或 CSS 方案。
- 禁止 emoji。
- 图标风格需统一，不混用多套视觉语言。

### 3. 卡片
- 卡片圆角、边框、阴影统一。
- 统计卡片更轻、更干净，突出数值和次要说明。
- 工具栏卡片和列表卡片做区分，但属于同一视觉系统。

### 4. 表单与筛选栏
- 输入框、下拉、按钮统一高度和圆角。
- 筛选工具栏要更像“业务操作台”，而不是普通堆叠控件。
- 页面按钮层级统一：主按钮 / 次按钮 / 危险按钮。

### 5. 表格
- 表头、行高、hover、空状态统一。
- 操作列按钮轻量化，不要过于突兀。
- 表格容器和分页区风格统一，避免块状拼接感。

### 6. 弹窗
- 标题区、正文区、底部操作区分层明确。
- 保持商务后台风格，不做花哨动效。

## 页面级任务拆分

### A. 共享层
- 收口 `AdminLayout.vue`
- 收口 `style.css` 中的共享后台视觉系统
- 统一导航图标、顶栏、品牌区、基础控件视觉

### B. 商品管理页
- 对齐预览稿的统计卡片、筛选区、列表区、分页区
- 去掉演示感文案，改成正式后台语气

### C. 券模板管理页
- 统一与商品页相同的页面骨架
- 优化弹窗表单层级与信息密度

### D. 用户券页
- 收口发券入口、导入结果、详情弹窗、二维码弹窗
- 去掉“联调/模拟”感知文案

### E. 核销中心页
- 收口头图、输入区、结果区、说明区
- 文案表达改成正式业务核销中心，不强调测试或联调语境

## 多 Agent 拆分
- Agent 1：共享布局与全局视觉系统
- Agent 2：商品管理 + 券模板管理
- Agent 3：用户券 + 核销中心
- 主 Agent：定义基线、整合实现、统一验收、构建验证、更新日志

## 验收标准
- 前端构建通过：`npm run build`
- 侧栏无 emoji 图标
- 页面无明显“模拟”字样
- 四个重点页面的卡片、筛选栏、表格、弹窗风格明显统一
- 整体观感接近 `Docs/ui/admin-preview-v1.html`，不再是简单 CRUD 页面气质

## 本轮输出
- 共享 UI 基线收口
- 重点页面 V2 视觉升级
- UI 工作日志更新
