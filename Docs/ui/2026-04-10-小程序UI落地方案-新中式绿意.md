# 发券小程序 UI 落地方案（新中式绿意）

## 1. 结论
- 小程序前端视觉基线正式采用 Stitch 项目 `projects/14906607459695872276` 中的 `新中式绿意` 方案。
- 作为开发基准的页面共有 4 张：
  - `首页 - 新中式绿意`
  - `商城 - 新中式绿意`
  - `我的券包 - 新中式绿意`
  - `个人中心 - 新中式绿意`
- 本方案适合当前“品牌零售 + 发券权益 + 券包购买 + ERP 核销”的业务定位，气质稳定，适合长期产品化。

## 3. 视觉语言

### 3.1 气质定位
- 关键词：东方留白、轻奢零售、柔和绿意、低饱和金饰感、品牌型会员权益。
- 不做强促销电商感，不走大红大紫高刺激路线。
- 首页与商城都应保留“品牌橱窗感”，不是纯工具列表。

### 3.2 色彩方向
- 主色建议：低饱和青绿 / 墨绿。
- 辅助色建议：米白、浅砂金、淡灰绿。
- 强调色只用于：主按钮、券状态标签、少量关键数字。
- 错误态和禁用态需收敛，避免破坏整体东方气质。

### 3.3 版式规则
- 顶部 Banner、专题区、卡片区之间保留明显留白。
- 卡片以大圆角、轻阴影、弱边界为主，不使用厚重描边。
- 模块标题使用“标题 + 副说明/筛选入口”的结构，增强运营感。
- 页面信息层级遵循：品牌头图 > 主权益入口 > 卡片列表 > 辅助运营区。

### 3.4 图标与插图
- 禁止 emoji。
- 图标统一采用线性/简洁填充风格。
- 商品图、券包图、活动图尽量用高质感实拍或品牌素材，不用廉价促销图。

## 4. 页面映射

### 4.1 首页
对应 Stitch：`首页 - 新中式绿意`

建议承载的业务模块：
- 顶部品牌头图 / 当前主活动 Banner
- 新人券专区
- 热门券包推荐
- 指定商品券 / 满减券分区入口
- 活动专题横滑卡片
- 推荐商品区
- 底部 TabBar：首页 / 商城 / 卡包 / 我的

首页目标：
- 第一屏先传达品牌与活动，而不是先展示表单或筛选器。
- 把“领券”和“买券包”都放在首页完成首轮触达。

### 4.2 商城
对应 Stitch：`商城 - 新中式绿意`

建议承载的业务模块：
- 搜索框
- 券包推荐区
- 商品列表 / 商品专题区
- 可按分类或活动标签切换
- 券包卡片支持：名称、价格、包含权益、适用说明、购买按钮

商城目标：
- 既承担“卖券包”，也承担“商品+券联动展示”。
- 避免纯商品货架感，保留“权益商城”特征。

### 4.3 我的券包
对应 Stitch：`我的券包 - 新中式绿意`

建议承载的业务模块：
- 顶部会员/用户摘要
- 券状态切换：可用 / 已使用 / 已过期
- 券卡片展示：券名称、优惠信息、有效期、适用门店、使用门槛
- 卡片点击进入券详情
- 支持展示“购买所得券”和“领取所得券”

页面目标：
- 核心是让用户快速找到“现在能用的券”。
- 视觉上要突出卡券本身，而不是列表容器。

### 4.4 个人中心
对应 Stitch：`个人中心 - 新中式绿意`

建议承载的业务模块：
- 用户头像 / 昵称 / OpenId 绑定后的用户摘要
- 我的订单
- 使用记录 / 核销记录
- 我的券包入口
- 门店说明 / 客服 / 规则 / 关于
- 如后续接企业微信或公众号能力，可在此处放消息订阅、服务入口

页面目标：
- 做轻运营个人中心，不做复杂会员中心。
- 你已经明确“会员体系不重”，所以这里不堆等级成长值。

## 5. 业务必补页面
当前 4 张 Stitch 基准页还不足以支撑完整业务，需要补以下页面：

### 5.1 券详情 / 核销二维码页
必须新增。

页面内容：
- 券名称
- 优惠规则
- 有效期
- 适用门店
- 适用商品 / 商品范围
- 核销二维码
- 券码明文（可选）
- 使用说明

这是 ERP 核销的核心页，优先级最高。

### 5.2 券包详情 / 购买确认页
必须新增。

页面内容：
- 券包封面
- 券包名称
- 售价
- 包含券明细
- 使用说明
- 购买按钮
- 支付结果跳转

### 5.3 订单结果页
建议新增。

页面内容：
- 支付成功 / 失败状态
- 订单号
- 发放结果摘要
- 跳转我的券包
- 返回商城

### 5.4 领券结果 / 活动详情页
可选。

如果首页活动比较重，建议补一个活动专题详情页，用于承接专题 Banner 点击。

## 6. 组件拆分建议
优先沉淀以下 Uni-app 组件：
- `BrandHeroBanner`：品牌头图 / 活动头图
- `CouponPackCard`：券包卡片
- `CouponCard`：卡券卡片
- `ProductCard`：商品卡片
- `SectionHeader`：模块标题 + 更多入口
- `SegmentTabs`：可用/已用/过期切换
- `BottomTabBar`：底部导航
- `QrCouponPanel`：二维码券详情面板
- `EmptyStateBlock`：空状态

## 7. Uni-app 页面目录建议
建议页面结构：
- `pages/home/index`
- `pages/mall/index`
- `pages/coupon-pack/detail`
- `pages/coupon/list`
- `pages/coupon/detail`
- `pages/profile/index`
- `pages/order/result`

如后续增加门店、规则、客服，可补：
- `pages/store/list`
- `pages/rule/index`
- `pages/service/index`

## 8. 开发顺序建议

### 第一阶段：壳与主页面
- 首页
- 商城
- 我的券包
- 个人中心
- 底部导航与全局主题变量

### 第二阶段：关键业务闭环
- 券详情 / 二维码页
- 券包详情 / 购买页
- 订单结果页

### 第三阶段：细节补强
- 空状态
- 加载骨架
- 状态标签
- 商品与券联动展示
- 规则页 / 说明页

## 9. 前后端对接注意点
- 券状态字段要直接服务于 UI 分组：可用 / 已使用 / 已过期。
- 券详情接口必须返回：模板名、券类型、有效期、门店范围、商品范围、二维码信息。
- 券包详情接口必须返回：券明细列表与封面素材。
- 首页接口建议拆成聚合数据：Banner、新人券、热门券包、推荐商品。
- 图片全部走素材体系，不再走零散 URL 字段方案。

## 10. 最终执行结论
- 小程序 UI 基线正式采用 `新中式绿意`。
- 当前可以立即开始 Uni-app 前端开发。
- 开发优先级以“4 张基础页面 + 3 张业务补页”为第一轮范围。
- 后续如需补视觉精修，可继续基于同一 Stitch 项目追加页面，而不再重新换风格。
