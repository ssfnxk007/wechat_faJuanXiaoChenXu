后台信息架构选择

这一步只决定左侧导航怎么重新分组和排序，不改任何路由、不改任何功能入口，只改变“人看到后台之后怎么理解它”。

A. 按业务链路组织

从“配置”到“投放”到“履约”一路排下去，后台会更像一套完整的发券运营工作台。

1. 首页总览仪表盘
2. 发券配置券模板、券包、券包明细、轮播图、小程序主题
3. 用户与资产用户管理、用户券
4. 交易与履约订单管理、核销中心
5. 内容与流量分享追踪
6. 基础资料门店管理、商品管理
7. 系统权限后台账号、角色、菜单

B. 按对象类型组织

分类更中性，越做越大也不容易乱，但业务主线会弱一些。

1. 首页总览仪表盘
2. 用户域用户管理、用户券
3. 商品门店域商品管理、门店管理
4. 券资产域券模板、券包、券包明细
5. 交易域订单管理、核销中心
6. 增长域轮播图、分享追踪、小程序主题
7. 系统权限后台账号、角色、菜单

当前建议：推荐 A. 按业务链路组织。

因为这个项目不是通用 ERP，它是“发卷业务后台”。把“发券配置 → 用户与资产 → 交易与履约 → 内容与流量”串起来，使用者更容易理解系统主线。
