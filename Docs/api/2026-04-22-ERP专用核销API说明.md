# ERP 专用核销 API 说明

## 配置

- 后端配置节点：`ErpApi`
- 默认请求头：`X-Api-Key`
- 示例：

```json
"ErpApi": {
  "ApiKey": "FaJuan-Erp-Key-2026-ChangeMe",
  "ApiKeyHeaderName": "X-Api-Key"
}
```

## 1. 预检接口

- 方法：`POST /api/erp/coupons/preview`
- 请求头：`X-Api-Key: <通用Key>`

请求体：

```json
{
  "siteCode": "S001",
  "couponCode": "CPN202604220001"
}
```

说明：

- 仅返回券信息与是否可核销
- 不修改券状态
- `settlementType=discount` 表示金额券
- `settlementType=product_redeem` 表示商品兑换券

响应示例：

```json
{
  "code": 200,
  "message": "可核销",
  "data": {
    "siteCode": "S001",
    "storeId": 1,
    "storeName": "南京东路店",
    "couponCode": "CPN202604220001",
    "userCouponId": 1001,
    "appUserId": 2001,
    "couponTemplateId": 3001,
    "couponTemplateName": "满40减5",
    "templateType": 4,
    "status": 1,
    "settlementType": "discount",
    "discountAmount": 5,
    "thresholdAmount": 40,
    "effectiveAt": "2026-04-22T10:00:00",
    "expireAt": "2026-05-22T23:59:59",
    "canWriteOff": true,
    "message": "可核销",
    "productScope": []
  },
  "success": true
}
```

商品券响应中的 `productScope` 会返回可兑换商品列表，供 ERP 选择最终核销商品。

## 2. 核销接口

- 方法：`POST /api/erp/coupons/writeoff`
- 请求头：`X-Api-Key: <通用Key>`

请求体：

```json
{
  "siteCode": "S001",
  "couponCode": "CPN202604220001",
  "selectedProductCode": "ERP001",
  "operatorName": "张三",
  "deviceCode": "POS-01"
}
```

说明：

- 必须由 ERP 在金额结算 / 出库成功后调用
- 普通优惠券可不传 `selectedProductCode`
- 商品券：
  - 若只配置了一个商品，可不传，后端自动选中
  - 若配置了多个商品，必须传 `selectedProductCode`

核销成功响应示例：

```json
{
  "code": 200,
  "message": "核销成功",
  "data": {
    "userCouponId": 1001,
    "couponCode": "CPN202604220001",
    "appUserId": 2001,
    "couponTemplateId": 3001,
    "couponTemplateName": "指定商品券",
    "siteCode": "S001",
    "storeId": 1,
    "storeName": "南京东路店",
    "settlementType": "product_redeem",
    "selectedProductCode": "ERP001",
    "selectedProductName": "洗护礼盒",
    "message": "核销成功"
  },
  "success": true
}
```

## 3. 当前规则

- 门店通过 `siteCode -> Store.Code` 匹配
- 券最终核销对象始终是 `UserCoupon`
- 商品券核销成功时会同时把履约状态更新为 `Fulfilled`
- 核销记录会额外记录所选商品 `ProductId`
