# 发券联调验收清单

日期：`2026-04-10`

## 目标
- 验证“手动发券”与“CSV 导入发券”主链路可用。
- 验证导入场景支持 `AppUserId / 手机号 / 小程序 OpenId / 公众号 OpenId` 四类用户标识。
- 验证关键失败分支：模板不存在、模板停用、用户不存在、超过每人限领、CSV 格式错误。

## 前置条件
- 后端可运行，数据库连接到 `FaJuanDb`
- 后台账号可进入“用户券”页面
- 已有可用 `AppUser` 测试数据
- 已有至少 2 个券模板：
  - 一个启用模板
  - 一个停用模板

## 手动发券验收

### 用例 1：正常发券
- 输入：有效 `couponTemplateId`、有效 `appUserIds`、`quantityPerUser=1`
- 预期：
  - 返回 `successCount=1`
  - `items[0].success=true`
  - `UserCoupon` 新增 1 条记录

### 用例 2：用户不存在
- 输入：不存在的 `appUserId`
- 预期：
  - `items[0].success=false`
  - `items[0].message=用户不存在`

### 用例 3：模板停用
- 输入：停用模板 ID
- 预期：
  - 接口失败
  - 提示 `券模板已停用，不能发券`

### 用例 4：超过每人限领
- 输入：对同一用户重复发券直到超限
- 预期：
  - 返回失败项
  - 提示包含 `超过每人限领`

## CSV 导入发券验收

### 支持的表头
- `appUserId,couponTemplateId,quantityPerUser`
- `mobile,couponTemplateId,quantityPerUser`
- `miniOpenId,couponTemplateId,quantityPerUser`
- `officialOpenId,couponTemplateId,quantityPerUser`

### 样例 1：按手机号导入
```csv
mobile,couponTemplateId,quantityPerUser
13800138000,3,1
13900139000,3,2
```

### 样例 2：按小程序 OpenId 导入
```csv
miniOpenId,couponTemplateId,quantityPerUser
mini-openid-001,3,1
mini-openid-002,3,1
```

### 样例 3：省略数量列，走页面默认数量
```csv
mobile,couponTemplateId
13800138000,3
13900139000,3
```

### 用例 1：正常导入
- 预期：
  - 返回 `parsedUserCount` 正确
  - `successCount` 与成功用户数一致
  - `invalidRows` 为空或仅包含故意错误行

### 用例 2：用户匹配失败
- CSV 中放一个不存在的手机号/OpenId
- 预期：
  - 行进入 `invalidRows` 或结果中显示失败
  - 提示 `未匹配到用户`

### 用例 3：模板 ID 与表单不一致
- CSV 填不同 `couponTemplateId`
- 预期：
  - 该行进入 `invalidRows`
  - 提示 `券模板ID无效或与表单不一致`

### 用例 4：数量非法
- CSV 中填 `0` 或负数
- 预期：
  - 该行进入 `invalidRows`
  - 提示 `发放数量无效`

### 用例 5：模板停用
- 使用停用模板导入
- 预期：
  - 接口失败
  - 提示 `券模板已停用，不能发券`
